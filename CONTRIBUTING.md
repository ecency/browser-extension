# Contributing to Hive Keeper

Thanks for taking the time to contribute. The community looks forward to your contributions.

If you don't have time to contribute, the following also help:

- Star the repo on GitHub
- Mention the project at local meetups or on social media
- Tell other Hive users about it

## Code of Conduct

This project is governed by the [Hive Keeper Code of Conduct](/CODE_OF_CONDUCT.MD). By participating, you are expected to uphold it.

## Asking a question

Before asking, search existing [issues](https://github.com/ecency/browser-extension/issues). If your question hasn't been raised, open a new issue with:

- A clear title
- As much context as you can provide
- Project and platform versions when relevant (Node.js, npm, browser)

## Reporting bugs

> **Security note:** never report security-sensitive issues in public. Email them to the address listed in [CODE_OF_CONDUCT.MD](/CODE_OF_CONDUCT.MD).

A good bug report includes:

- A reproduction (sequence of steps + expected vs. actual behavior)
- Stack trace if available
- OS, browser, and extension version
- Screenshots or short screen recordings if helpful

Open the bug at https://github.com/ecency/browser-extension/issues/new.

## Suggesting enhancements

For new features or improvements, open an issue at https://github.com/ecency/browser-extension/issues with:

- A descriptive title
- A step-by-step description of the proposed enhancement
- Why it would be useful and to whom
- Optional screenshots or mockups

Search existing issues first to avoid duplicates.

## Submitting code

1. Fork the repo and create a topic branch off `master`.
2. Read the [README](/README.md) for build instructions.
3. Follow the existing code style (TypeScript, React, conventional commit messages).
4. Add tests where reasonable. Run `npm test` before opening a PR.
5. Open a pull request against `master` with a clear description of the change and any related issue numbers.
